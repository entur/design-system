## Komponenter

### DatePicker

<ImportStatement imports="DatePicker" />
<Props of={DatePicker} />

Komponenten vår er bygget på `react-datepicker`, og støtter dermed en rekke props, selv om 100% kompabilitet ikke kan garanteres.
Les mer om disse [her.](https://reactdatepicker.com/)

Den støtter også `date-fns` og det anbefales at datofunksjonene fra `date-fns` brukes.
Eksempelvis `addWeeks` for å sette en maksgrense på én uker man kan velge datoer fra.

<Playground>
  {() => {
    const [date, setDate] = React.useState();
    return (
      <DatePicker
        label="Velg dato innen én uke"
        style={{ width: '15rem' }}
        selectedDate={date}
        onChange={date => setDate(date)}
        minDate={new Date()} //Mindate, Fra og med
        maxDate={addWeeks(new Date(), 1)} // Til og med
      />
    );
  }}
</Playground>

Komponenten kan også brukes som kun kalender uten input eller motsatt. For kun kalender benytter man `inline`-propen og for kun input uten kalender bruker man `hideCalendar`-propen.

<Playground>
  {() => {
    const [date, setDate] = React.useState();
    const [date2, setDate2] = React.useState();
    return (
      <>
        <DatePicker
          inline // Gjør at kun kalender vises
          label="Velg dato"
          style={{ width: '15rem' }}
          selectedDate={date}
          onChange={date => setDate(date)}
        />
        <DatePicker
          hideCalendar // Gjør at kun input vises
          hideCalendarButton
          style={{ width: '15rem', marginTop: '1rem' }}
          selectedDate={date2}
          onChange={date => setDate2(date)}
          label="Velg dato"
        />
      </>
    );
  }}
</Playground>

For deg som ønsker å støtte mobil, så eksisterer det også en `NativeDatePicker`.
Den wrapper en `input` hvor `type="date"`, som i noen sammenhenger kan være og foretrekke.
